# The Synthetic Data Factory: Using Nemotron to Generate Its Own Training Data

Over 98% of the data used to align NVIDIA's Nemotron models is synthetically generated. Not curated by humans. Generated by the models themselves.

That number deserves a pause. The model that generates your answers was trained on data that another model created. And it works.

This repo implements that pipeline. Generate synthetic training data. Score it automatically. Filter for quality. Export for fine-tuning.

## The Pipeline

The architecture is straightforward. Two models. Four steps.

**Step 1 — Generate.** A Nemotron instruct model takes a topic and produces diverse question-answer pairs. You specify the domain (customer support, medical, legal, technical, finance, education) and the model generates realistic examples that match the domain's tone and complexity.

**Step 2 — Score.** Each generated pair is evaluated by the Nemotron Reward model across five dimensions:

- **Helpfulness** — Is this actually useful to the person asking?
- **Correctness** — Is the information accurate?
- **Coherence** — Does the answer flow logically?
- **Complexity** — Is the depth appropriate for the question?
- **Verbosity** — Is the length appropriate?

Each dimension gets a numeric score. The reward model was trained specifically for this — it grades LLM outputs the way a human evaluator would, but at machine speed.

**Step 3 — Filter.** Pairs below a configurable quality threshold are discarded. A threshold of 1.5 removes the bottom tier. A threshold of 2.5 keeps only genuinely good pairs. You control the tradeoff between quantity and quality.

**Step 4 — Export.** Passing pairs are written as JSONL in the standard fine-tuning format — each line is a conversation with user and assistant messages. Drop the file into any fine-tuning pipeline.

## Why This Works

The insight is that generating data is cheap. Scoring data is cheap. The expensive part was always human evaluation. The reward model replaces the human evaluator.

This creates a flywheel:

1. Generate 1000 pairs for a domain
2. Score them automatically
3. Keep the top 200
4. Fine-tune a smaller model on those 200
5. The fine-tuned model is now better at that domain
6. Repeat

NVIDIA used this exact pattern to train the Nemotron-4 340B family. They generated synthetic data with the instruct model, scored it with the reward model, and used the filtered output to align the next generation. The synthetic data was better than the curated human data it replaced.

## The Five Scoring Dimensions

The reward model does not just return a single "good/bad" score. It evaluates along five orthogonal dimensions. This matters because a response can be correct but unhelpful, or helpful but incoherent.

**Helpfulness** captures whether the response addresses what the user actually needs. A technically accurate but off-topic answer scores low on helpfulness.

**Correctness** is factual accuracy. The reward model catches hallucinations, wrong numbers, and logical errors. This is the dimension that most directly maps to trust.

**Coherence** measures logical flow. Does the answer contradict itself? Are the paragraphs in a sensible order? Does it stay on topic?

**Complexity** is about depth matching. A simple factual question should get a simple answer. A nuanced analytical question should get a detailed response. Both extremes — too shallow and too deep — score low.

**Verbosity** penalises both extremes. Too terse means important context is missing. Too verbose means the answer is padded with filler. The sweet spot depends on the question.

## Domain Templates

The pipeline includes pre-built domain templates that steer generation toward realistic scenarios:

- **Customer Support** — Complaints, product questions, returns, billing, troubleshooting
- **Medical** — Symptoms, treatments, prevention, when to seek care
- **Legal** — Contracts, rights, procedures, compliance
- **Technical** — Debugging, architecture, best practices, code review
- **Finance** — Budgeting, investing, taxes, accounting, planning
- **Education** — Study techniques, curriculum design, assessment, engagement

Each template adds domain-specific instructions to the generation prompt. The result is data that sounds like it came from that domain, not generic Q&A.

## Seed Examples

You can provide seed examples to control the style and format of generated data. The model uses your examples as a reference for:

- Question length and complexity
- Answer depth and structure
- Tone and formality
- Technical vocabulary level

Three seed examples are usually enough to establish a consistent style across the entire generated dataset.

## What You Can Build With This

**Domain-specific fine-tuning data.** Generate 500 pairs about your product, score them, fine-tune a smaller model. The result is a model that knows your domain without needing to be trained on proprietary documents.

**Evaluation datasets.** Generate diverse test cases for your LLM. The reward scores give you a baseline to compare against.

**Data augmentation.** Supplement a small human-curated dataset with synthetic pairs. The scoring ensures the synthetic data meets the same quality bar as the human data.

**Curriculum for RLHF.** The reward scores are exactly what RLHF needs — preference signals across multiple dimensions. Generate pairs, score them, rank them, use the rankings as preference data.

## The Numbers

NVIDIA's own results:

- 98% of alignment data was synthetically generated
- The synthetic data pipeline produced higher-quality training data than human curation
- The reward model grades responses on a 0-4 scale across all five dimensions
- At scale, this is orders of magnitude cheaper than human evaluation

The models are available via the NVIDIA NIM API. No GPU required. No model hosting. Just API calls.
